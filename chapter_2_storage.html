Storage

Graph mapping to key value pair

  1. Vertex

Key: [type(1), partId(3), vertexId(4), tagId(4), version(8)]
Value: [property1, property2, ... propertyN]

  1. Edge

Directed edge

Positive edge:

Key: [type(1), partId(3), srcId(4), edgeType(4), Ranking(4), dstId(4), version(8)]
Value: [property1, property2, ... propertyN]

Negative edge:

Key: [type(1), partId(3), dstId(4), edgeType(4), Ranking(4), srcId(4), version(8)]
Value: [property1, property2, ... propertyN]

This positive/negative edge for partition-friendly that the one directed edge will be stored in the partition contains source vertex and partition destination vertex.

  1. Key Space

[spaceId, partId, key]

In nebula, we can determine a key by the [spaceId, partId, key] triad.

The spaceId is used to separate the key.

The partId is used to identify the partition of raft group.

The key is used to identify the one record in the actual storage engine.
